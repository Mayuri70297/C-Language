Basic Structure of C Program


Structure of C Program is defined by set of rules called protocol, to be followed by programmer while writing C program. All C program are having sections/parts which are mentioned below -

  • Documentation section
  • Link section
  • Definition section
  • Global declaration section
  • Functio prototype declaration section
  • Main function
  • User defined function definition section

    • Many programming language such as java, C++, Pascal, COBOL etc., have a certain standard format which should be followed of the programmer. In the same way, the C language also has the standard structure which resembles this.

    #include<Header Files>
    [Global Declaration]
    main()
    {
    <Local Declaration> ;
    <Statements> ;
    }

    • In the syntax, we mentioned include header files, which means we have to include the header file in the second line, you can see global declarations. It indicates that a function or the variable an be declared here and accessed anywhere in the program.
    • The main function, this is neccessary in each C program, because the execution starts in main function.
    • Each C Program must be started with the main function.
    • Following main function, we used the curly braces which are parts to start and of closing of the program.
    • The statements are nothing but, the goal of the function.

    Understanding Header Files

    We need to declare all functions in C language before you can use them. Because it would be helpful to group related function declaration and manage them in a single place. This is the function of header files.
    Header files are vital in large projects because they give you an overview of source code without having to look through every line of code.
    The header files are the collection of pre defined functions neccessary for the operation of a prgram if it in a section written by the user of code, or as an element of the standard libraries of C language by conventio, the names of the header files finish with .h(dot h).
    The suffix of .h is used by header files which are equipped with operating system. We can also make our own header files if we need.

    The header files define the Following functions :

    • Structures of certain files and subroutines.
    • Type the synonyms or implementation chaeacteristics.
    • Constants and macros which are substituted during the phase of pretreatment of language C.

    Header Files

    • string.h - This file contains the functios which are related to string manupulations.
    • alloc.h - It has the function which are related with memory allocation.
    • time.h - This header file consists of the time related functions.
    • conio.h - The con io.h has console (i.e. monitor) input output functions.
    • ctype.h - It include the functions which are related to conversion.
    • stdlib.h - It has standard library functions and also memory allocation.
    • graphics.h - It has many built in functions that support graphics.
    • dos.h - It defines many functions that are related to date and delay which is used to set interval.
    • process.h - It contains system supporting functions.

    The Main Function

    A C program composed of one or more functions.
    A functions is a line of one or more statements which are joined in curly brackets carry our a certain operations and return a simple value to the program in which they reside; it also could be look like subroutine.
    All the C program must have main function. The execution of program start with main function.
    This mai function will call usually other functions coming from same program, and other of the libraries to carry out its work.
    Mainly function has two parts :

    • Declaratory Part :It handles declaration of the variables.
    • Achievable Part : It contain the code which carries out of detail charge.
